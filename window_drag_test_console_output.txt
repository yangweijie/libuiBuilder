Window组件拖拽测试 - 控制台输出
==========================================

测试时间: 2025-12-17

测试步骤:
1. 打开 designer.html
2. 清空画布
3. 拖拽 Window 组件到空画布
4. 验证组件结构

控制台输出:
[LOG] designer对象存在: true @ :2
[LOG] 组件数量: 1 @ :1
[LOG] 组件 0: {id: component_1, type: window, parent: undefined, childrenCount: 0} @ :3
[LOG] 组件数量: 1 @ :5
[LOG] 组件 0: {id: component_1, type: window, parent: undefined, childrenCount: 0} @ :7

HTML预览代码:
<!DOCTYPE html> 
<ui version="1.0"> 
  <window title="窗口" size="800,600" centered="true" margined="true"> 
    窗口内容 
  </window> 
</ui>

测试结果分析:
✅ 组件数量: 1 (正确，应该是1个)
✅ 组件类型: window (正确)
✅ 父组件: undefined (正确，根组件没有父组件)
✅ 子组件数量: 0 (正确，刚拖拽的Window组件没有子组件)
✅ HTML结构: 只有一个window元素，没有嵌套 (修复成功!)

结论: Window组件拖拽bug修复成功！
当拖拽Window组件到空画布时，直接创建了一个Window组件，
而不是创建嵌套的Window组件。
